## 组件树与数据流

```
App.tsx
  └─ useTimer() → { timeLeft, mode, totalTime, isRunning, progress, start, pause, reset }
       ├─ TimerDisplay  ← { timeLeft, mode, progress }
       │    └─ ProgressRing  ← { progress, mode } (工作=红色 #ef4444, 休息=蓝色 #3b82f6)
       └─ ControlButtons ← { isRunning, onStart, onPause, onReset }
```

App 是唯一调用 useTimer 的地方，所有子组件通过 props 接收数据。

**数据流方向**: 单向 — useTimer 持有所有状态，App 将数据通过 props 向下传递给子组件。子组件通过回调（onStart, onPause, onReset）向上通知 App，App 调用 useTimer 的方法来修改状态。

## 类型定义

```typescript
// src/types.ts
export type TimerMode = 'work' | 'break'

export interface TimerState {
  mode: TimerMode
  timeLeft: number       // 当前倒计时秒数
  totalTime: number      // 当前模式总时长（秒），用于计算进度
  isRunning: boolean
  progress: number       // 进度 0-1
}

// useTimer 返回类型（组合 TimerState + 操作方法）
export type UseTimerReturn = TimerState & {
  start: () => void
  pause: () => void
  reset: () => void
}
```

## useTimer Hook 设计

```typescript
// src/hooks/useTimer.ts
参数: 无
返回: UseTimerReturn = { timeLeft, mode, totalTime, isRunning, progress, start, pause, reset }

内部逻辑:

状态:
  mode: 'work' | 'break'  初始 'work'
  timeLeft: number         初始 WORK_DURATION（1500）
  isRunning: boolean       初始 false
  lastTick: number | null  上次 tick 的时间戳 (Date.now())
  
常量:
  WORK_DURATION = 25 * 60  // 1500 秒
  BREAK_DURATION = 5 * 60  // 300 秒

useEffect:
  if (!isRunning) return
  
  lastTick = Date.now()
  const id = setInterval(() => {
    const now = Date.now()
    const elapsed = Math.floor((now - lastTick) / 1000)
    if (elapsed >= 1) {
      lastTick = now  // 重置基准时间戳
      setTimeLeft(prev => {
        if (prev <= 1) {
          // 先返回 0，显示 00:00 一帧
          if (prev === 1) return 0
          // 已经是 0 了 → 切换到新模式
          return switchMode()  // switchMode 返回新模式时长，避免闭包读取旧 mode
        }
        return prev - elapsed  // 扣除实际经过的秒数
      })
    }
  }, 200)  // 高频检查 (200ms)，保证 1 秒精度
  return () => clearInterval(id)  // cleanup: 组件卸载时清理

start():
  if (isRunning) return  // 幂等守卫: 防止快速双击或已运行时重复启动
  if (timeLeft <= 0) setTimeLeft(currentModeDuration())  // 倒计时已结束则重置
  setRunning(true)
  lastTick = Date.now()

pause(): setRunning(false)

reset():
  setTimeLeft(currentModeDuration)
  setRunning(false)

switchMode():
  const nextMode = mode === 'work' ? 'break' : 'work'
  setMode(nextMode)
  return nextMode === 'work' ? WORK_DURATION : BREAK_DURATION
  // 注意: switchMode 不改变 isRunning — 自动切换后保持运行
  // 返回新模式的时长，避免闭包陷阱（读取旧的 mode 值）

currentModeDuration():
  return mode === 'work' ? WORK_DURATION : BREAK_DURATION

进度计算 (派生值，非独立状态):
  progress = 1 - (timeLeft / totalTime)
  totalTime = currentModeDuration()
```

### 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 返回类型 | 扁平结构 | 符合 React Hooks 惯例（如 useState），调用方解构方便 |
| 计时策略 | Date.now() 时间差 | 浏览器后台 tab 会节流 setInterval，时间差方式不受影响 |
| 检查频率 | 200ms | 比 1000ms 更频繁，确保不会因节流丢失整秒 |
| 00:00 可见 | 显示一帧 | prev=1 → 返回 0（显示 00:00），下一个 tick → 切换模式 |
| 自动切换后 | 保持运行 | 切换模式不改变 isRunning，实现无缝循环 |
| 快速双击开始 | 幂等守卫 | start() 内部 if (isRunning) return |
| 组件卸载 | cleanup | useEffect 返回 clearInterval，防止内存泄漏 |
| 依赖管理 | 明确清单 | 测试依赖：vitest, @testing-library/react, @testing-library/jest-dom, jsdom |
| CSS 方案 | Tailwind CSS | 原子化 CSS，快速构建一致 UI，无需维护独立样式文件 |
| 色彩系统 | 见下方色彩规范 | 页面背景、文字、进度环、按钮颜色完整定义 |
| 排版系统 | 见下方排版规范 | 倒计时/标签/按钮字号、字重、间距 |
| 响应式 | 见下方响应式策略 | 移动端适配：进度环缩放、字体调整 |

## 响应式策略

进度环尺寸随视口自适应：
- 桌面端 (≥768px): `size = 280px`
- 移动端 (<768px): `size = min(240px, 65vw)`

实现方式：使用 CSS `clamp()` 或 Tailwind 响应式前缀 `md:size-[280px] size-[min(240px,65vw)]`。

倒计时数字随视口缩放：桌面 `text-7xl`，移动 `text-5xl`。

## 计时精度策略

使用基于 `Date.now()` 的时间差计算，而非依赖 `setInterval` 固定间隔。浏览器后台 tab 会节流 `setInterval`，导致计时漂移。

实现方式：`useEffect` 中以 200ms 间隔高频检查，每次比较当前时间戳与上次记录的 `lastTick` 时间戳差值，从 `timeLeft` 中扣除实际经过的秒数。

**模式切换时机**: 当 `timeLeft` 降到 1 时，先返回 0（显示 00:00 一帧），下一个 tick 检测到 timeLeft === 0 时执行模式切换。这样用户能短暂看到 00:00，视觉上确认"时间到了"。

## SVG 进度环规格

```typescript
// Props
interface ProgressRingProps {
  progress: number       // 0-1，已消耗比例
  mode: TimerMode        // 决定颜色
  size?: number          // 默认 280
  strokeWidth?: number   // 默认 8
}

// 实现关键参数
center = size / 2
radius = center - strokeWidth
circumference = 2 * Math.PI * radius
strokeDashoffset = circumference * (1 - progress)  // 从顶部(12点)顺时针填充
// transform: rotate(-90 center center) — SVG 默认从3点方向开始，旋转-90度从12点开始
// 颜色: work → #ef4444 (红), break → #3b82f6 (蓝)
// 背景环: #334155 (深灰，配合深色背景)
// strokeLinecap: round
// 动画: transition: stroke-dashoffset 0.5s linear (进度平滑过渡)
//       transition: stroke 0.3s ease (颜色平滑切换)
// 无障碍: role="progressbar", aria-valuenow={timeLeft}, aria-valuemin=0, aria-valuemax={totalTime}, aria-label="剩余时间"
```

## 倒计时格式化

```typescript
function formatTime(totalSeconds: number): string {
  const minutes = Math.floor(totalSeconds / 60)
  const seconds = totalSeconds % 60
  return `${String(minutes).padStart(2, '0')}:${String(seconds).padStart(2, '0')}`
}
```

## 测试策略

使用 Vitest + React Testing Library，覆盖：

- `useTimer.test.ts`: Hook 状态转换逻辑
  - 初始状态：mode='work', timeLeft=1500, isRunning=false
  - 开始 → isRunning=true, timeLeft 开始递减
  - 暂停 → isRunning=false, timeLeft 保持不变
  - 恢复 → isRunning=true, 从暂停时的 timeLeft 继续
  - 归零 → timeLeft=0 → 下一帧自动切换到 break 模式, timeLeft=300
  - 休息归零 → 自动切换回 work 模式, timeLeft=1500
  - 重置 → timeLeft 回到当前模式初始值, isRunning=false
  - 快速双击开始 → 第二次调用无效果（幂等性）
  - 自动切换后 isRunning 保持 true
  - 组件卸载 → 计时器被清理（无内存泄漏）
- `TimerDisplay.test.tsx`: 组件渲染
  - 时间格式化显示（25:00, 05:00, 00:00 等）
  - 模式标签显示（🍅 工作时间 / ☕ 休息时间）
  - 进度环渲染（验证 progress prop 正确传递）
- `ControlButtons.test.tsx`: 按钮显示与交互
  - isRunning=false → 显示"开始"按钮 + "重置"按钮
  - isRunning=true → 显示"暂停"按钮 + "重置"按钮
  - 点击"开始" → 调用 onStart
  - 点击"暂停" → 调用 onPause
  - 点击"重置" → 调用 onReset
